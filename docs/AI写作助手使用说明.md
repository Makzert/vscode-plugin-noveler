# AI 写作助手使用说明

这份文档只讲一件事：**怎么把 Noveler 的 AI 写作助手真正用起来。**

---

## 1. 使用前准备

先确认你的项目里已经配置好 AI：

打开 `novel.jsonc`，找到 `noveler.ai`：

```jsonc
"ai": {
  "baseUrl": "https://api.openai.com/v1",
  "apiKey": "你的 Key",
  "model": "gpt-4.1-mini",
  "models": {
    "high": "gpt-4.1",
    "medium": "gpt-4.1-mini",
    "low": "gpt-4.1-nano"
  }
}
```

说明：

- `high`：正文生成、扩写、改写、润色
- `medium`：总结、人物信息归纳
- `low`：人名识别等低成本任务

如果没有配置 `models`，插件会自动回退到默认 `model`。

---

## 2. 两种使用方式

### 方式 A：侧边栏使用

适合第一次上手。

1. 打开左侧 `Noveler` 侧边栏
2. 找到 `AI 写作助手`
3. 选择模式
4. 输入补充要求
5. 点击生成，或在输入框里按 `Ctrl/Cmd + Enter`
6. 预览结果后再插入 / 替换 / 追加

### 方式 B：编辑器内使用

适合日常高频写作。

你可以**不离开编辑器区域**，直接通过快捷键完成主流程。

---

## 3. 推荐工作流（纯键盘）

### 工作流 1：续写当前内容

1. 光标放在要续写的位置
2. 按 `Ctrl/Cmd + Alt + M` 切到合适模式
3. 按 `Ctrl/Cmd + Alt + Enter` 开始生成
4. 在编辑器里看流式预览
5. 满意后按：
   - `Ctrl/Cmd + Alt + I` 插入
   - `Ctrl/Cmd + Alt + R` 替换
   - `Ctrl/Cmd + Alt + P` 追加

### 工作流 2：给 AI 一个额外要求

例如你想告诉 AI：

> 这一段要压抑一点，动作更利落，别写解释性旁白

操作：

1. 在编辑器里选中这句要求
2. 按 `Ctrl/Cmd + Alt + S`
3. 这段文字就会被保存为 AI 的补充要求
4. 回到正文位置，按 `Ctrl/Cmd + Alt + Enter` 生成

### 工作流 3：切换模式

反复按 `Ctrl/Cmd + Alt + M`，可以轮换：

- 续写当前内容
- 改写选区 / 段落
- 扩写选区 / 段落
- 润色对话
- 总结当前章节

### 工作流 4：切换应用目标

反复按 `Ctrl/Cmd + Alt + T`，可以轮换：

- 插入光标处
- 替换选区 / 段落
- 追加到文末

---

## 4. 快捷键总表

- `Ctrl/Cmd + Alt + A`：打开 AI 写作助手
- `Ctrl/Cmd + Enter`：在侧边栏输入框中生成
- `Ctrl/Cmd + Alt + Enter`：在编辑器里直接生成
- `Ctrl/Cmd + Alt + I`：应用结果（插入）
- `Ctrl/Cmd + Alt + R`：应用结果（替换）
- `Ctrl/Cmd + Alt + P`：应用结果（追加）
- `Ctrl/Cmd + Alt + S`：将选中文本设为 AI 补充要求
- `Ctrl/Cmd + Alt + M`：切换 AI 模式
- `Ctrl/Cmd + Alt + T`：切换 AI 应用目标
- `Ctrl/Cmd + Alt + Backspace`：清空 AI 补充要求
- `Ctrl/Cmd + Alt + D`：一键丢弃当前预览
- `Ctrl/Cmd + Alt + /`：打开当前预览的 Diff 对比
- `Ctrl/Cmd + Alt + K`：根据本章内容同步本章人物

---

## 5. 现在的预览是怎么工作的

生成时有两处反馈：

- **编辑器内联预览**：方便你不用移开视线
- **侧边栏预览**：方便看完整结果和警告信息

说明：

- 编辑器里的预览只是“预览”，**不会直接改正文**
- 只有你执行插入 / 替换 / 追加后，结果才会真正写入文档
- thinking / 分析内容会在预览阶段就被隐藏，不应直接显示给你

---

## 6. 人物档案会发生什么

现在人物同步改为**手动触发**。

当你想根据当前章节内容更新人物时：

- 按 `Ctrl/Cmd + Alt + K`
- 或执行命令 `Noveler: 同步本章人物`

执行后，插件会：

- 自动回填章节 Front Matter 里的 `characters[]`
- 自动识别人名
- 必要时自动创建最小人物档案
- 自动补全部分人物信息

当前策略比较保守：

- 会自动创建 / 补全
- **不会自动删除人物档案**

这样做是为了避免误删已有设定。

---

## 7. 为什么“我按了没反应”

优先检查这几项：

1. 当前打开的是不是 Markdown 文件
2. `novel.jsonc` 里的 `noveler.ai.apiKey / baseUrl / model` 是否已配置
3. 快捷键是否和你的 VS Code / 系统快捷键冲突
4. 侧边栏里是否已经保存过模式 / 提示要求

---

## 8. 推荐上手方式

如果你第一次用，建议这样试：

1. 打开一个章节 Markdown
2. 在正文里写两三段内容
3. 选中一句要求，比如“保持压抑氛围，减少解释”
4. 按 `Ctrl/Cmd + Alt + S`
5. 按 `Ctrl/Cmd + Alt + Enter`
6. 观察编辑器里的流式预览
7. 看章节标题上方的 CodeLens，直接点 `接受 AI 预览 / 丢弃 AI 预览 / Diff 对比`
8. 如果本章出现了新人物，按 `Ctrl/Cmd + Alt + K`

这样最容易体会这套工作流。

---

## 9. 你应该怎么理解这套 AI

Noveler 当前的 AI 不是“全自动替你写完”的模式。

更适合这样理解：

- 你负责创作方向
- 插件负责快速生成、改写、润色、总结
- 你在编辑器内快速确认和应用

也就是：**以手写为主，AI 在旁边加速。**
